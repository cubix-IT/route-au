SEASONAL INTELLIGENCE — THE CORE IDEA
====================================================
The landing page, destination cards, hero copy, and
Surprise Me algorithm all shift with the season.
No user input needed. The app just knows.

SEASONS (Southern Hemisphere):
  Summer  Dec–Feb   Hot Melbourne, 35–42°C. Go coastal.
  Autumn  Mar–May   Best season. Mild, golden, harvest.
  Winter  Jun–Aug   Cold, grey. Go cosy and inland.
  Spring  Sep–Nov   Warming up. Wildflowers, blossom.

  function getCurrentSeason() {
    const m = new Date().getMonth() + 1
    if ([12,1,2].includes(m))  return 'summer'
    if ([3,4,5].includes(m))   return 'autumn'
    if ([6,7,8].includes(m))   return 'winter'
    return 'spring'
  }

SURPRISE ME — SEASONAL + WEATHER ALGORITHM
====================================================
User taps "Surprise Me" → no other input needed.

STEP 1: Get context
  - Browser geolocation (or default Melbourne CBD)
  - Current time → trip length:
      Friday 3pm–midnight  → 2-day trip
      Saturday 6am–midday  → 1-day trip
      Saturday midday–5pm  → half-day, back by evening
      Sunday               → short loop only
  - Current season → base ranking

STEP 2: Score all clusters
  score = seasonal_score[season] × 10
        + weather_bonus           (sunny, mild → +20)
        + distance_bonus          (closer → slight bonus)
        - rain_penalty            (rain >70% → -30)
        - heat_penalty            (temp >38°C → -20 for inland)

  Weather fetched from Open-Meteo for each cluster centre.
  Parallel fetch, cached 4 hours in IndexedDB.

STEP 3: Present result
  "It's a perfect autumn day in the Yarra Valley"
  "22°C · Mostly sunny · 1hr 10min from Pascoe Vale"
  Full itinerary loads below immediately.
  [ Try another escape ] button shows #2 ranked cluster.

SEASONAL SURPRISE EXAMPLES:
  January Saturday morning:
    → "Head to Portsea. 26°C, light breeze, perfect beach day."

  April Friday evening:
    → "Bright is peak colour right now. Go tonight, back Sunday."

  July Saturday:
    → "Hepburn Springs is calling. Misty, 12°C, and the
       outdoor baths are perfectly timed. Here's your weekend."

  October Sunday morning:
    → "Grampians wildflowers are at peak. Day trip, back by 6pm."


WEATHER ON EVERY CARD
====================================================
Real-time weather badge on every destination card.
Open-Meteo, free, already in codebase.

FETCH STRATEGY:
  - On landing page load: fetch weather for top 8 clusters
    in parallel (all within same Open-Meteo multi-location call)
  - Cache: IndexedDB, 4-hour TTL
  - Display: "22°C · Sunny" or "14°C · Showers likely"
  - If rain >70%: card gets subtle grey overlay + "Best avoided
    this weekend" note (still shown, just deprioritised)
  - If temp >38°C (summer, inland): "Very hot this weekend —
    head to the coast instead" suggestion

VENUE DATA STRATEGY
====================================================
No overthinking. Build as we go. Improve over time.

STARTER SET: ~15 venues per cluster = ~180 venues total

DATA SOURCES:
  Google Places API — one-time batch script
    Pulls: description, rating, review_count, phone,
           price_level, opening_hours
    Cost: ~$2-3 total for 180 venues. Run it once.

  Wikipedia — for national parks, nature spots, walking
    tracks (free, good text, just grab the first paragraph)

  Manual override — you add the important detail
    ("the wood-fired sourdough is the reason to come")
    wherever Google's description is weak
